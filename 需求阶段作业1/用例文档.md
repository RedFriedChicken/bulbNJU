##  3、详细用例描述
###  用例1、商品库存管理

|     ID      | 1                                        |
| :---------: | :--------------------------------------- |
|   **名称**    | 商品库存管理                                   |
|   **创建者**   | 宋金泽                                      |
|  **创建日期**   | 2017/9/11                                |
| **最后一次更新者** | 宋金泽                                      |
| **最后更新日期**  | 2017/10/16                                |
|   **参与者**   | 库存管理人员                                   |
|  **触发条件**   | 有商品的变动或需要对商品进行查看                         |
|  **前置条件**   | 管理人员已经被录入，可以登录系统                         |
|  **后置条件**   | 商品信息变动被记录                                |
|   **优先级**   | 中                                        |
|  **正常流程**   | 1.系统显示商品分类信息。包括分类名称，分类编号。<br>2.库存管理人员根据分类列表选择商品分类。对商品分类的改动包括增加分类、删除分类、修改分类。<br>3.管理人员确认分类选择。<br>4.系统保存分类修改或显示分类下子分类。<br>库存管理人员可重复2 ~ 5步直到系统显示出叶节点。<br>5.系统显示出叶节点分类下的所有商品。商品的属性有：编号、名称、型号、库存数量、进价、零售价、最近进价、最近零售价。<br>6.库存管理人员可根据商品列表选择增加商品、删除商品、修改商品和查询商品进行相关操作。<br>7.库存管理人员保存完成商品信息的查看修改。<br>8.系统保存商品信息修改记录。增加的商品可选作为“入库”，删除的商品可选作为“出库”。<br>库存管理人员重复4~5步并返回2步重复直到完成商品信息的所有改动。<br>9.库存管理人员确认操作。10.系统返回根目录。 |
|  **扩展流程**   | 3a.增加一个分类时，系统会自动给该分类生成一个编号。<br>3b.删除一个分类时，若分类下的子分类或叶节点为空，则直接删除对应的商品分类。若分类下的子分类或叶节点不为空，则可选择将分类下的子分类合并到其他分类，或选择新建分类，或选择手动修改对应子分类的父节点。<br>2a.库存管理人员输入内容不符合增删改的格式时，系统显示错误，输入无效。<br>3a.管理人员可取消确认，系统返回上层目录。<br>5a.商品编号根据其所在的分类以及添加次序自动生成。商品的查询可以通过输入关键字、商品编号等进行模糊查找。 <br>6a.库存管理人员查询时输入内容不符合查找格式或查找无效时，系统返回信息未找到符合条件的商品。<br>9a.库存管理人员可选择取消确认操作，系统不保存任何修改信息并返回根目录。|
|  **特殊要求**   | 1.商品分类是树状结构，且只有叶节点的分类才能添加商品，一旦分类下有商品就不能在该分类下再添加子分类。<br>2.因为在实际中进价和售价一直都在变化，所以一开始规定一个进价和售价作为默认值，保存一个最近的进价和售价属性。<br>3.当商品分类名字发生变化时，对应所有子商品的所属分类名均发生变化。 |
###  用例2、库存分析

|     ID      | 2                                        |
| :---------: | ---------------------------------------- |
|   **名称**    | 库存分析                                     |
|   **创建者**   | 宋金泽                                      |
|  **创建日期**   | 2017/9/11                                |
| **最后一次更新者** | 宋金泽                                      |
| **最后更新日期**  | 2017/10/16                               |
|   **参与者**   | 库存管理人员                                   |
|  **触发条件**   | 库存商品存在问题；库存统计需要；有库存变动                    |
|  **前置条件**   | 库存管理人员信息被录入，可以登录系统。                      |
|  **后置条件**   | 库存相关报单可以提交打印。                            |
|   **优先级**   | 中                                        |
|  **正常流程**   | 1.0库存查看<br>1.库存管理人员选择查看库存，并选择查看库存的时间段。<br>2.系统搜索并显示此时间段内的出/入库数量/金额，销售/进货的数量/金额。库存数量要有合计。<br>3.库存管理人员点击查看完毕。<br>4.系统返回上一界面。<br>1.1库存盘点<br>1.库存管理人员选择库存盘点。<br>2.系统自动盘点当天的库存，包括当天的各种商品的名称，型号，库存数量，库存均价，批次，批号，出厂日期，并且显示行号。<br>3.库存管理人员查看库存列表。<br>&emsp;3a. 库存管理人员选择导出Excel<br>&emsp;&emsp;1.系统生成表格并导出Excel<br>1.2库存报单<br>1.库存管理人员选择库存报单<br>2.系统显示可供填写的报单<br>3.库存管理人员选择并填写单据<br>&emsp;3a.库存管理人员选择库存赠送单<br>&emsp;&emsp;1.库存管理人员根据赠送商品，填写库存赠送单。库存赠送单内容包括日期、商品名称、商品编号、商品对应分类、赠送商品数量。<br>&emsp;3b.库存管理人员选择库存报溢单<br>&emsp;&emsp;1.库存管理人员根据实际库房中多出商品，填写库存报溢单。库存报溢单内容包括日期、商品名称、商品编号、商品对应分类、系统库存数量、实际数量、溢出数量。<br>&emsp;3c. 库存管理人员选择库存报损单<br>&emsp;&emsp;1.库存管理人员根据实际库房缺失的商品，填写库存报损单。库存报损单内容包括日期、商品名称、商品编号、商品对应分类、系统库存数量、实际数量、缺少数量。<br>&emsp;3d. 库存管理人员选择库存报警单<br>&emsp;&emsp;1.库存管理人员设置警戒数量，当库存数量低于该数值时进行提示。并填写库存报警单。库存报警单内容包括日期、商品名称、商品编号、商品对应分类、系统库存数量、警戒数量。<br>4.库存管理人员点击提交。<br>5.系统自动提交库存单据<br>6.单据完成审批<br>7.系统自动从库存中减去这些商品。 |
|  **拓展流程**   | 1.0.1a.库存管理人员输入时间段无效（后段比前段时间靠前、两时间相同或两时间未发生等），系统显示提示输入非法。<br>1.0.2a.库存管理人员输入时间段有效，但时间段内无任何信息，系统显示未找到符合要求的库存记录。<br>1.1.2a.若系统当天无库存商品的信息（详见1.1.2），则系统返回当前信息为空。<br>1.2.3.3d.1a.库存管理人员填写库存报警单时，该商品警戒高于库存中实际数量时，系统提示输入信息非法。<br>1.2.4a.库存管理人员提交时若未完成，系统提示有部分信息未完成。<br>
|  **特殊要求**   | 1.系统中任何单通过审批，就自动完成相应数据的更改。               |
